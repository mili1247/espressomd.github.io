58 /** Integrate equations of motion
59  * @param n_steps Number of integration steps, can be zero
60  * @param reuse_forces Decide when to re-calculate forces:
61  * - -1: recalculate forces unconditionally
62  * (mostly used for timing)
63  * - 0: recalculate forces if @ref recalc_forces is set,
64  * meaning it is probably necessary
65  * - 1: do not recalculate forces (mostly when reading
66  * checkpoints with forces)
67  *
68  * @details This function calls two hooks for propagation kernels such as
69  * velocity verlet, velocity verlet + npt box changes, and steepest_descent.
70  * One hook is called before and one after the force calculation.
71  * It is up to the propagation kernels to increment the simulation time.
72  *
73  * This function propagates the system according to the choice of integrator
74  * stored in @ref integ_switch. The general structure is:
75  * - if reuse_forces is zero, recalculate the forces based on the current
76  * state of the system
77  * - Loop over the number of simulation steps:
78  * -# initialization (e.g., RATTLE)
79  * -# First hook for propagation kernels
80  * -# Update dependent particles and properties (RATTLE, virtual sites)
81  * -# Calculate forces for the current state of the system. This includes
82  * forces added by the Langevin thermostat and the
83  * Lattice-Boltzmann-particle coupling
84  * -# Second hook for propagation kernels
85  * -# Update dependent properties (Virtual sites, RATTLE)
86  * -# Run single step algorithms (Lattice-Boltzmann propagation, collision
87  * detection, NpT update)
88  * - Final update of dependent properties and statistics/counters
89  *
90  * High-level documentation of the integration and thermostatting schemes
91  * can be found in doc/sphinx/system_setup.rst and /doc/sphinx/running.rst
92  *
93  * @return number of steps that have been integrated
94  */
95 int integrate(int n_steps, int reuse_forces);
