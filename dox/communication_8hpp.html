Detailed Description

This file contains the asynchronous MPI communication.

It is the header file for communication.cpp.

The asynchronous MPI communication is used during the script evaluation. Except for the master node that interprets the interface script, all other nodes wait in mpi_loop() for the master node to issue an action using mpi_call(). mpi_loop() immediately executes an MPI_Bcast and therefore waits for the master node to broadcast a command, which is done by mpi_call(). The request consists of a callback function and two arbitrary integers. If applicable, the first integer is the node number of the slave this request is dedicated to.

To add new actions (e.g. to implement new interface functionality), do the following:

  • write the mpi_* function that is executed on the master
  • write the mpi_*_slave function
  • register your slave function with one of the REGISTER_CALLBACK macros

After this, your procedure is free to do anything. However, it has to be in (MPI) sync with what your new mpi_*_slave does. This procedure is called immediately after the broadcast with the arbitrary integer as parameter.
