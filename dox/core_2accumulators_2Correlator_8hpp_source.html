19 /** @file
20  *
21  * This module computes correlations (and other two time averages) on
22  * the fly and from files.
23  *
24  * The basic idea is that the user can write arbitrary functions A and B that
25  * can depend on e.g. particle coordinates or whatever state of the MD box.
26  * These functions can be vector-valued, always indicated by dim_A and dim_B.
27  *
28  * The way they can be correlated is can be formulated as a (vector-valued)
29  * operation on A and B. One example would be to calculate the product
30  * component by component, and if A and B are both the particle velocities,
31  * then one would obtain
32  * @f[ <v_1^x v_1^x(t)>, <v_1^y v_1^y(t)>, <v_1^z v_1^z(t)>,
33  * <v_2^x v_2^x(t)>, <v_2^y v_2^y(t)>, <v_2^z v_2^z(t)>, \ldots @f]
34  *
35  * The idea of the algorithm is to not keep all As and Bs in memory, but
36  * feed the algorithm (or the container object) successively by new As and Bs
37  * and new correlation estimates are calculated on the fly. Those As and Bs
38  * which have exceeded a certain age are first compressed (see below) and then
39  * discarded.
40  *
41  * To save memory, increase statistics and make the calculation possible for
42  * many orders of magnitude in time, the order-<i>n</i> blocking algorithm in
43  * @cite frenkel02b (algorithm 9, section 4.4.2 p. 95)
44  * is used. Thus not all As and Bs of the whole "past" are stored but
45  * some of them are blocked. In this implementation, a blocking based on 2 is
46  * always applied: all As and Bs not older than a certain @c tau_lin are stored
47  * as they were, those which are older are stored as average values. All As
48  * and Bs older than <tt>2*tau_lin</tt> are compressed in blocks of four, etc.
49  *
50  * This leads to a hierarchical "history": on level 0 the last @c tau_lin values
51  * are stored. This is done in a cyclic array: the newest is always appended
52  * at the end of the array, and if the array length is reached, values
53  * are appended at the beginning, overwriting older values. We therefore
54  * have to carry the index <tt>newest[0]</tt> which tells, what is the index
55  * of the newest value of A and B.
56  *
57  * As soon as the first row of As and Bs is full, two of them are
58  * compressed by calculating the arithmetic mean, and stored in the
59  * first element on level 1. Then we can overwrite these two values
60  * on level 0, because we have saved them. If necessary,
61  * we produce space on level 0 by compressing to level 1. On
62  * level 1 an array with @c tau_lin entries is available to store
63  * the level-1-compressed values. It is successively filled
64  * and also cyclic. When it is filled, the values are stored
65  * on level 2 and so on.
66  *
67  * This allows to have a "history" over many orders of magnitude
68  * in time, without the full memory effort.
69  *
70  * Correlations are only calculated on each level. For
71  * <tt>tau=1,2,..,tau_lin</tt> the values are taken from level 1.
72  * For <tt>tau=tau_lin, tau_lin+2, .., 2*tau_lin</tt> we take the values
73  * from level 2. On level 2 we also have values for <tt>0,..tau_lin-2</tt>,
74  * but these are discarded as we have "better" estimates on level 1.
75  *
76  * The functions A and B can get extra arguments. This can e.g. be an
77  * integer describing the "type" of the particles to be considered,
78  * or in case of files, it is a file_data_source pointer, which tells
79  * the function from where to read. These arguments have to be a single
80  * pointer to an object that carries all relevant information
81  * to obtain A and B (from the MD Box or somewhere else).
82  *
83  * The correlation has to be initialized with all necessary information, i.e.
84  * all function pointers, the dimensions of A and B and their dimensions, etc.
85  *
86  *
87  * TODO: There is a lot of stuff to do:
88  * - Expand the file_data_source so that one can specify which
89  * columns of the file are to be processed
90  * - calculate an estimate of average values. This might be
91  * even necessary to calculate
92  * @f$\left<\left(A(\tau)-\left<A\right>\right)\left(B(\tau)-\left<B\right>\right)\right>@f$,
93  * which is often probably what people want
94  * - Use the A_args to calculate As and Bs only for particular
95  * particle types (especially and example, so that other people can follow)
96  * - Use the A_args to calculate molecular stuff in combination with
97  * the topology concept
98  * - Write a destructor
99  */

120 /** The main correlator class
121  *
122  * Data organization:
123  * We use a ring-like way to manage the data: at the beginning we have a
124  * linear array, which we fill from index 0 to @c tau_lin. The index
125  * <tt>newest[i]</tt> always indicates the latest entry of the hierarchic
126  * "past" For every new entry in is incremented and if @c tau_lin is reached,
127  * it starts again from the beginning.
